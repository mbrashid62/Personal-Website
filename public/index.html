Temperature Reader

This was an academic team project for my Senior Design class. Overall this project was challenging but my group and I worked very well together and were successful in accomplishing it.

The project consisted of three distinct components. An Arduino circuit board for reading temperature from a sensor, a Java program for collecting serialized data from the board, and a web application for displaying the temperature readings in real-time.

The latter two components was what I worked on. This involved importing the Java RXTX library to handle I/O from the Arduino and programming Java Swing components to perform the desired operations.

For the web interface, I used jQuery to parse the incoming data stream from plaintext to JSON format. This enabled me to use Morris (a Javascript chart library) to visualize the data in real-time by refreshing the data every second.

In the end it was a very cool feeling see all the components of the lab work together. We could dip the temperature sensor in ice water and watch the chart fall and begin breathing on it and see the chart rise up again!

  • Primary technologies used: Java (RXTX Library), Arduino, and JavaScript (Morris and jQuery)

Sail

Sail is a full-stack side project of mine that allows users to explore the vast ocean of Soundcloud tracks. The app prompts users to enter their favorite genres and returns some sounds they might like from the Soundcloud API.

Most of my side projects have involved primarily front-end code. I wanted to build something with back-end functionality because I believe all web developers (even front-end ones) should have some skill with back-end code.

I taught myself Angular and Firebase, two google products with steep learning curves, but tremendous potential when the light bulb clicks.

In my app, I used Firebase to assist me with handling user login and registration. Firebase also grants me to ability to allow users to save sounds they like to a NoSQL database -- which consequently, allows users to view their sounds later.

For the front-end, I followed an MVC pattern using Angular. By following several tutorials and practicing on my own, I was able to design front-end that I believe is pretty scalable and robust.

The main goal of this side project was to familiarize myself with new front-end and back-end technologies. By doing so I became a more well-rounded developer. What was interesting from this project was that learning Angular from a React background helped me understand the nuances of both frameworks at a deeper level. Likewise, learning how a NoSQL data structure works compared to a traditional MySQL data structure helped me understand the different ways app store data from a new perspective.

As I grow as an engineer, I will be continuing to work on side projects because I believe they are the best way to learn new skills.

  • Primary technologies used: Angular, Firebase, and the Soundcloud API
  • Running Demo: Sail
  • Source: The GitHub repo

MIDI Drums

For my final project for my Embedded Systems class, I created a 4 piece electric drum set. This project involved both a hardware and a software component.

Using an arduino as my microcontroller, I designed a circuit that transmits MIDI data to my computer when a user taps a drum pad (tupperware container). The drum pads are triggered by four individual piezo sensors (sensors that detect vibration) attached beneath the lid of each container. One for bass, snare, hi-hat, and tom.

My arduino code initializes a MIDI connection to my computer and transmits the appropriate MIDI bytes. As MIDI bytes are read, the corresponding MIDI note is sent to my comptuer's MIDI channel, triggering an output as if the user was playing any off-the-shelf MIDI instrument. The result can be heard through any software that outputs MIDI notes (Garageband, Logic, Abelton, etc).

The inspiration for this project came from my difficulty finding a drummer to jam with as a guitar player. Unlike guitars, drums are expensive, loud, and difficult to move around. I wanted to build a portable system my friends and I can use to play easy, simple drum beats with to our guitars.

This project was a great way to apply what I learned about embedded systems in a practical and fun way!

  • Primary technologies used: Arduino, C, and several hardware components

Encrypto Chat

This chat app utilizes the Socialist Millionaire Protocol to verify the identity of two end users through the use of a common, shared secret.

The project works by encrypting each users' public and private keys as well as their shared secret (which they are prompted to enter right away). By performing a series of complex math sequences, the app can then verify if the user is indeed who they say they are, allowing users to exchange encrypted messages safely!

When using this app, it is helpful to have a console window open to see the exchange of fmessages betwen client and server. The OTR module I used makes all this data available to the client - this includes the authentication process as well as the message exchanges.

Please be patient while waiting for authentication, it may take several minutes for the SMP algorithm to complete.

My inspiration for this app came from wanting to solidify my understanding of the client server relationship in web applications. I believe at a fundamental level, chat apps are a great way to do this. In addition, I discovered a lot of interesting and useful information while reading about encryption and message security.

  • Primary technologies used: Javscript - (socket.io, otr, and other node modules)
  • Source: Encrypto Chat
